# Getting Support

Thanks for using OpenKOS. This page explains where to go depending on what you need. Please pick the right channel — it helps us keep things organized and respond well.

> **Note:** OpenKOS is **pre-alpha**. There is no released version yet and no formal support commitments or response-time guarantees. The channels below are best-effort and community-driven.

## I have a question or want to discuss an idea

Use **[GitHub Discussions](https://github.com/jasonssdev/openkos/discussions)**. This is the place for:

- questions about how OpenKOS works or is meant to be used;
- ideas you want to talk through before opening a formal proposal;
- sharing what you're building or how you use the project.

Please search existing discussions first — your question may already be answered.

## I found a bug

Open a **[bug report issue](https://github.com/jasonssdev/openkos/issues/new/choose)** using the bug template. Include your OS, Python version, how you installed OpenKOS, and the smallest steps that reproduce the problem. Search existing issues first to avoid duplicates.

## I want to request a feature or propose a design change

- For a concrete capability, open a **feature request** issue.
- For anything that touches the knowledge model, OKF conformance, the pipeline, or public interfaces, open a **design proposal / RFC** issue.

Both use templates available from the [new issue chooser](https://github.com/jasonssdev/openkos/issues/new/choose).

## I found a security vulnerability

**Do not open a public issue.** Follow the private disclosure process in **[SECURITY.md](SECURITY.md)** (GitHub private vulnerability reporting or email).

## I want to contribute

See **[CONTRIBUTING.md](CONTRIBUTING.md)** for how to get involved, set up your environment, and open a pull request, and **[GOVERNANCE.md](GOVERNANCE.md)** for how decisions are made.

## Documentation

Start with the [README](README.md), then the design docs in [`docs/`](docs/): the [vision](docs/vision.md), the [knowledge object model](docs/knowledge-object-model.md), the [roadmap](docs/roadmap.md), and the [tech stack](docs/tech_stack.md).
