# Design: `openkos init` — create an OKF workspace

## Technical Approach

`init` is a **check-everything-then-write** command over four collaborating modules. Behaviour is split so that the format seam (`model/okf.py`), the bundle tree (`bundle/`), and the workspace root (`config.py`) each own exactly one thing, and `cli/main.py` only sequences them and maps outcomes to exit codes. No `engine.py`, no `llm/`, no async, no Protocol — none has code that arrives in this slice.

## Module map

| Module | Owns | Never knows |
|---|---|---|
| `model/okf.py` | The **format**: `OKF_VERSION = "0.1"`, `RESERVED = {"index.md", "log.md"}`, frontmatter load/dump, §9 conformance (rules 1–2). | Workspaces, `raw/`, `openkos.yaml`. |
| `bundle/index.py` | The **bytes of `index.md`**: `render_index() -> str` — frontmatter from `okf.py` + empty body. Pure. | The filesystem. |
| `bundle/log.py` | The **bytes of `log.md`**: `render_log(today: date) -> str` — no frontmatter (reserved, `docs/okf-alignment.md:38`). Pure. | The filesystem, the clock. |
| `bundle/bundle.py` | The **bundle root**: `create(bundle_dir: Path, today: date)` — mkdir + write both reserved files. The only module that writes inside the OKF tree. | The workspace root above it. |
| `config.py` | The **workspace root**: `WorkspaceLayout`, `is_workspace(root)`, `refusal_reason(root)`, `write_config(root, name, model)`, `write_agents(root)`. `is_workspace` and `refusal_reason` both read one private generator (`_refusal_conditions`) so the "already a workspace" conditions have exactly one definition. | Frontmatter, OKF. |
| `cli/main.py` | Typer `app` + the `init` command: pre-flight, sequence, exit codes, stdout. | Format and layout details. |
| `src/openkos/templates/` | Package **data** (not a package — no `__init__.py`): `AGENTS.md`, `openkos.yaml`. | — |

**Where "workspace" lives.** A bundle is not a workspace: `raw/` sits *outside* it, and that is the load-bearing decision (`docs/architecture.md:141-154`). `bundle/bundle.py` therefore cannot own the workspace — it would have to reach above its own root. `config.py` owns it, because `openkos.yaml` **is** the workspace: it is the marker whose presence defines one (Q7.6), and it declares the layout (`raw: raw/`, `bundle: bundle/`, `docs/cli.md:104-106`). `AGENTS.md` goes there too — both are workspace-root files that belong to the engine, not the format (`examples/good-life-demo/AGENTS.md:12`).

> **Named friction**: `config.py` writing `AGENTS.md` reads oddly. It is the least-bad home inside the package set the proposal fixed. When `ingest` needs the same composition, extract `workspace.py` (or `engine.py`) then — not now, on spec (`openspec/config.yaml:44`).

## Architecture Decisions

| # | Decision | Alternatives rejected | Rationale |
|---|---|---|---|
| D1 | **Two-phase init**: Phase A pure reads, Phase B writes. No interleaving. | Write-then-rollback. | The refusal guarantee ("exit 1, write nothing") holds *by construction*, not by cleanup. |
| D2 | **Exclusive-create mode (`"x"`) on all four file writes.** | `"w"` + prior existence check. | Pre-flight is the UX; `"x"` is the *guarantee*. Closes the Phase-A→B TOCTOU window: a racing file causes `FileExistsError`, never a clobber. |
| D3 | **No cleanup path.** | `try/except` + remove what we made. | Deleting directories on an error path, in a cwd that may hold the user's notes and a pre-existing `raw/`, is a worse failure than a partial workspace. Marker (`openkos.yaml`) written **last**, so a crashed init never claims workspace status it cannot back. Partial-I/O-fault recovery is manual and named as a limitation. |
| D4 | **Templates are package data under `src/openkos/templates/`**, read via `importlib.resources.files("openkos") / "templates" / …`. | Python string constants; a `templates/` sub-package. | A 45-line `AGENTS.md` as a string constant is unreviewable and unlintable. `files("openkos") / "templates"` needs no `__init__.py`, so no empty scaffolding. **Packaging implication below.** |
| D5 | **`openkos.yaml` is generated by placeholder substitution on a template file** — not by a YAML dumper. | `ruamel.yaml` `CommentedMap`; PyYAML `dump`. | `docs/cli.md:97-109` and `examples/good-life-demo/openkos.yaml:7-10` both carry explanatory comments. No dumper *emits* comments (ruamel *round-trips* them — a different job we do not have). A template gives comment fidelity for free and byte-exact output. **Consequence: `ruamel.yaml` does NOT move to runtime in this slice**; it moves when `ingest` must round-trip a user-edited config. |
| D6 | **Stub `main()` is deleted, not aliased.** | Keep `main = app`. | The alias *is* the evidenced trap: `import openkos; openkos.main()` would import fine and exit non-zero under Click — green locally, red in CI. Deletion fails loudly at import, forcing `ci.yml` to change in the same commit. |
| D7 | **Zero ADRs.** See the gate below. | — | Neither candidate clears both conditions. |

## Runtime dependencies

| Package | Moves to runtime? | Why |
|---|---|---|
| `typer` | **Yes** | `cli/main.py` imports it. |
| `python-frontmatter` | **Yes** | `model/okf.py` parses frontmatter for §9 rules 1–2. |
| `ruamel.yaml` | **No** (D5) | Nothing reads or round-trips YAML in this slice. |
| `pydantic` | **No** | No structured LLM output here. |

    │                               │                                      │  AGENTS.md exists?

    │                               │

    │    exit 1  ── NOTHING WRITTEN ───┘                                  │

## Testing Strategy

| Layer | What | How |
|---|---|---|
| Unit (pure) | `render_index`, `render_log`, `render_config` | Plain asserts on strings. **No filesystem.** |
| Unit (fs) | `bundle.create`, `write_config`, `write_agents`, `check_conformance` | `tmp_path` + an explicit `Path` argument. **No mocking, no injection.** |
| Unit (cli) | 4 refusal branches, success, exit codes, stdout/stderr | `typer.testing.CliRunner` + `monkeypatch.chdir(tmp_path)`. |
| Packaging | Templates ship in the wheel | CI smoke step (above). |

**Injection: exactly one — the date.** `render_log(today: date)` takes it as a parameter, not a `Clock` Protocol (`AGENTS.md:37` reserves Protocol for *extension* points; a clock is not one). The caller passes `datetime.now().astimezone().date()` — the user's **local** calendar date, not UTC's. Ruff `DTZ` requires timezone-*awareness*, not UTC specifically; both `datetime.now(UTC).date()` and `datetime.now().astimezone().date()` satisfy it equally, so `DTZ` does not decide between them. The actual reason is that OpenKOS is local-first and single-user, and `log.md` is a person's chronological history read by a person — `## YYYY-MM-DD` is a heading someone scans, not a machine timestamp. Git sets the precedent: it records commit dates in local time with an offset, for the same reason. Everything else takes a `Path`; `tmp_path` is the whole seam. **No `--path` flag for testability** — that would smuggle in the argument Q7.6 deferred; `monkeypatch.chdir` covers it. `Path.cwd()` is read on exactly one line, in `cli/main.py`.

## Threat Matrix

N/A — no routing, shell, subprocess, VCS/PR automation, executable-file classification, or process-integration boundary. `init` writes four files under `Path.cwd()` and spawns nothing. (`git init` is deferred to `add-workspace-git`; the Ollama pull to `add-model-selection`.)

## ADR gate — evaluated, zero created

| Candidate | (1) Technology/pattern/interface/trade-off? | (2) Hard to reverse? | Verdict |
|---|---|---|---|
| Console entry `openkos.cli.main:app` | **Yes** — a public interface. | **No.** The proposal's own framing is conditional: "hard to reverse *once a wheel is published*" — and **no wheel has been published**. Version is `0.1.0`; `ci.yml` has three jobs (`quality`, `test`, `build`) and **no publish step**; no release exists. Reversal cost today = revert one commit across three files, which is the rollback plan verbatim. | **No ADR** |
| Removing model selection from `init`'s contract | **Yes** — a user-facing trade-off. | **No.** It is a *deferral*, named `add-model-selection`, and additive when it lands: init gains a prompt, the `model:` field already exists. Nothing must be un-done. | **No ADR** |

**Both also fail a prior test: neither is a decision *this design makes*.** `AGENTS.md:34` and `docs/architecture.md:106` already state the entry point "moves to `openkos.cli.main:app` when the `cli` package lands in MVP 1". `docs/tech_stack.md:108` already mandates shipping a working default until the spike runs. Design is **executing documented decisions, not making them** — an ADR here would record a decision taken elsewhere, months ago, with its forces no longer fresh.

Also weighed and rejected: `qwen3.5:9b` as default (one-line reversible *by design*, `docs/tech_stack.md:108`); templates-as-package-data (reversible); "idempotent by refusal, never overwrite" (a *behaviour contract* → it belongs in the `workspace-init` spec, which is exactly the line `AGENTS.md:60` draws).

`docs/adr/README.md:19` says the log "begins with the first decision taken while building MVP 1" — tempting to open it with `0001` for ceremony. Declined: `openspec/config.yaml:27` says **"When in doubt, do not create one."**

## Migration / Rollout

No data migration, no feature flags, no persisted state, no published artifact. Rollout is the 3-PR chain in the proposal; PR 1 is the only one that can break a green `main`.

## Gaps and contradictions found (reported, not silently resolved)

1. **`AGENTS.md` is not in the refusal set — but init writes it.** Proposal Q7.6 names only `openkos.yaml`, `raw/`, `bundle/`. Yet "never overwrites" is absolute, and a cwd containing an `AGENTS.md` is *highly* plausible (any repo already using agent tooling). Design adds it as a fourth pre-flight condition (D2's `"x"` mode would catch it anyway, but as an ugly traceback rather than a clean exit 1). **sdd-spec must cover this branch.**
2. **The rollback plan contradicts D6.** `proposal.md:94` says `src/openkos/__init__.py` "must keep its stub `main()` until PR 1's revert window closes." But rollback *is* `git revert` of the whole commit, which restores `main()` automatically. Keeping a dead stub adds a symbol nothing calls, that the 90% branch gate must cover, and that a later commit must remember to delete. **Recommend correcting that line at tasks time.**
3. **§9 rule 3 is not implemented.** `check_conformance` covers rules 1–2 only. Rule 3 (reserved files follow §6/§7 structure) needs concept documents to be meaningful; the proposal's own success criterion frames the check as *vacuous* at init. Deferred to `lint`.
4. **Quote style will diverge from the demo.** `python-frontmatter`'s dump emits `okf_version: '0.1'` (single-quoted); `examples/good-life-demo/bundle/index.md:2` shows `"0.1"`. Both are valid YAML parsing to the same string. **sdd-spec should assert the *parsed value*, not the bytes** — or this needs a literal template instead.
5. **The static `AGENTS.md` template describes folders init does not create.** Its tree (`examples/good-life-demo/AGENTS.md:22-25`) lists `sources/`, `concepts/`, `people/`, `decisions/` — all deferred by Q7.3. Acceptable: it is an operating *manual* describing the shape, and `ingest` creates them on first write. Noted, not changed.
6. **`ci.yml:97-98`'s comment is already inaccurate** — it claims the smoke test proves "py.typed ships with it", but `import openkos; openkos.main()` never checks `py.typed`. Fix the comment while editing the step.

## Line-budget impact

This design **pushes past the proposal's ~550–750 estimate, to roughly ~650–850**. Additions: the 45-line `AGENTS.md` template (D4), the `openkos.yaml` template, a fourth refusal branch plus its RED test (gap 1), and the second CI smoke step. The 3-PR chain still absorbs it; **PR 3 grows from ~200 to ~280**. The chain decision remains required before apply.
